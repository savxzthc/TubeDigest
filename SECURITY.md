# Security Policy

## Supported Versions

Security fixes are applied to the latest release on the default branch.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting feature for the repository, or contact the maintainers privately.

Include:

- A concise description and impact
- Reproduction steps or a minimal proof of concept
- Affected routes, versions, and configuration
- Suggested mitigation, if known

Never include live API keys, private transcript content, access tokens, or personal data. Maintainers should acknowledge a report within seven days and coordinate disclosure after a fix is available.
